UX Considerations

1

탭 전환 = URL 라우팅

각 탭은 별도 URL(/dashboard, /session, /history)로 관리하여 브라우저 뒤로가기·북마크 지원. 단, 해커톤 데모 특성상 클라이언트 상태 전환도 허용

2

Dashboard = 기본 랜딩

앱 최초 진입 시 Dashboard 탭으로. Active Session은 "New Session" 버튼 또는 세션 시작 시 자동 전환

3

History 마스터-디테일 패턴

좌측 리스트 + 우측 상세로 분할. 리스트에서 세션 클릭 시 우측이 전환되어 빠르게 탐색 가능

4

데이터 = 로컬 Mock

해커톤 데모이므로 세션 히스토리 데이터는 하드코딩된 Mock 데이터 사용. 실제 세션 저장은 스코프 외

5

Active 탭 인디케이터

현재 탭 하단에 보라색 언더라인으로 명확한 위치 표시. 기존 텍스트 컬러 변경 + 언더라인 추가

6

레이아웃 일관성

Dashboard·History 모두 동일한 헤더 + 스테이터스바 프레임 유지. 중앙 콘텐츠 영역만 교체
