  • ✖ "분석 완료" 후 다시 타겟 사이트로 돌아가려면 Start Capture부터 새로 시작해야 함 (화면 공유 권한 재요청)

  • ✖ 분석 완료 후 "sent" 상태에서 FlowBar가 완료 상태로 고정되어 다음 행동이 불명확
  • ✖ 실제로 MediaStream은 pause 상태로 살아있음 → 재활용 가능한데 버리고 있음

상세 동작 명세

1. "분석 완료" 클릭 시

  • 세션 저장 + Notion 전송 (기존 로직 유지)
  • FlowStep: "report" → "sent"
  • 사이드바: 로그 탭으로 전환 (기존 유지)
  • NEW: 성공 토스트 표시 — "리포트가 저장되었습니다"
  • NEW: 3초 카운트다운 시작

2. 3초 카운트다운 중 (ScreenFeed 영역)

  • ScreenFeed 영역에 "캡처 재개 중... 3s" 오버레이 표시
  • 카운트다운: 3 → 2 → 1
  • "새 세션 시작" 버튼 제공 (카운트다운 취소 → Stop Capture → idle)
  • 카운트다운 동안 비디오 프리뷰는 계속 표시 (스트림 살아있으므로)

3. 자동 복귀 (3초 후)

  • pausedRef.current = false + Worker 재시작
  • FlowStep: "sent" → "capturing"
  • FlowBar: 리셋되어 "캡처" 단계 활성화 (새 사이클)
  • 프레임 버퍼 초기화 (이전 세션 프레임 제거)
  • REC 인디케이터 + FPS/Latency 오버레이 복귀

4. 스트림이 끊어진 경우 (예외 처리)

  • 사용자가 분석 중 브라우저 화면 공유를 중단한 경우
  • streamRef.current === null 체크
  • 스트림 없으면 → idle 상태로 복귀 + "화면 공유가 종료되었습니다" 로그
  • "Start Capture" 버튼 표시 (기존 동작)

구현 포인트

1

useScreenCapture — resumeCapture 함수 추가

pauseCapture의 역순: pausedRef = false + Worker 재시작 + 프레임 버퍼 초기화

2

useDraftReport — 분석 완료 후 resume-capture 이벤트 발행

3초 setTimeout 후 window.dispatchEvent(new CustomEvent("resume-capture"))

3

ScreenFeed — resume-capture 이벤트 리스너

스트림 존재 확인 → resumeCapture() 호출 + flowStep을 "capturing"으로 전환

4

ScreenFeedView — 카운트다운 오버레이 UI

flowStep === "sent" 일 때 카운트다운 + "세션 종료" 버튼 표시
