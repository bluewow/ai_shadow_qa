# Antigravity Workflows Guide

이 디렉토리는 Antigravity가 프로젝트를 분석, 기획, डिजाइन 및 구현할 때 참고하는 커스텀 워크플로우 명세서입니다. 
채팅창에서 아래의 슬래시 명령어 형식(`/[명령어] [인자]`)으로 특정 역할의 AI 보조를 받을 수 있습니다.

## 🛠 주요 명령어 목록

### 1. 🔍 기획 및 아키텍처 (PM & Analysis)
- **`/pm-work [아이디어 또는 기능명]`**
  - **용도:** 새로운 기능을 기획할 때 사용합니다.
  - **동작:** 아이디어를 구체화하기 위해 몇 가지 핵심 질문을 던지고, 이를 바탕으로 `docs/idea/` 에 한 페이지 짜리 UI 기획서(HTML)를 예쁘게 만들어 줍니다.
- **`/ask [분석할 기능 또는 파일]`**
  - **용도:** 코드를 수정하기 전, 해당 기능이나 모듈의 현재 상태와 영향을 분석합니다.
  - **동작:** 시스템 구조를 파악하고, 작업 진행 전 명확히 짚고 넘어가야 할 점들에 대해 질문합니다.

### 2. ✏️ 개발 플래닝 및 구현 (Plan & Execution)
- **`/plan [개발 목표 내용]`**
  - **용도:** `/ask`로 분석이 끝낫거나 개발 목표가 명확할 때, 실제 구현에 앞서 작업 계획을 세웁니다.
  - **동작:** 추가/수정/삭제할 파일 목록과 Step-by-Step 작업 순서를 작성하여 승인을 대기합니다.
- **`/go` (또는 `/go [바로 구현할 내용]`)**
  - **용도:** 계획된 작업이나 명확한 요구사항을 실제 코드로 작성하고 반영합니다.
  - **동작:** 기존 파일 편집 및 생성을 진행하고 완료 후 작업 요약을 보고합니다.

### 3. 🎨 통합 디자인 보조 (Design Assistant)
- **`/design-work [대상 화면/이미지/가이드 요청]`**
  - **용도:** UI/UX 구조 개선 제안, 테마 추출, 스타일 가이드 생성을 전담합니다.
  - **동작 (요청에 따라 아래 3가지 중 하나로 동작):**
    1. **화면 UX 개선:** 기존 코드를 읽고 개선된 UI 목업을 `docs/redesign/` 폴더에 HTML로 제안 (A/B/C 탭 비교).
    2. **테마 추출:** 레퍼런스 이미지를 분석하여 컬러와 폰트를 딴 후 `docs/style/theme-candidates.html` 에 적용 시안 도출.
    3. **가이드 생성:** 현재 프로젝트의 Tailwind/CSS 설정과 컴포넌트 공통 패턴을 긁어모아 `docs/style/guide.md`와 `preview.html` 렌더링.

---

**Tip**: 한 번의 대화 내에서 다음과 같은 체인(Chain) 형태로 작업하면 높은 퀄리티의 결과물을 얻을 수 있습니다!
> 💡 `/pm-work` (기획/시각화) ➡️ `/ask` (사전 분석) ➡️ `/plan` (작업 계획) ➡️ `/go` (실제 코드 구현)
