# Go (구현)

**승인된 계획(Plan)을 바탕으로 실제 코드를 수정하고 구현합니다.**
Antigravity에서 `/go <구현할 기능>` 형식으로 사용하며, 앞서 세운 계획에 따라 진행합니다.

## 진행 방식

1. **컨텍스트 및 계획 재확인**
   - `implementation_plan.md` 내역이나 최근 대화의 계획 내용을 확인합니다. (필요시 파일 읽기 도구 사용)
   - 프로젝트 스타일 가이드나 설정이 있으면 그 규칙에 맞춰 코드를 작성합니다.

2. **코드 구현 (실제 도구 활용)**
   - `write_to_file`, `replace_file_content`, `multi_replace_file_content` 도구를 적극 활용하여 코드를 작성합니다.
   - 한 번 도구를 실행할 때마다 변경 사항이 적절히 반영되었는지 점검합니다. 

3. **진과정 트래킹 (Task Boundary)**
   - 작업 시작 시 `task_boundary` 도구를 활성화하고 `EXECUTION` 모드로 설정합니다.
   - 각 Step이 끝날 때마다 상태를 업데이트합니다.

## 중요 규칙
- 한 번 시작하면 계획된 범위에서 멈추지 않고 완료까지 진행합니다. (중대 결함이나 모호성이 발견될 때만 중단하고 보고)
- 기존 코드 패턴 및 린트(Lint) 규칙, 컴포넌트 컨벤션을 엄격히 준수합니다.
- UI 개발인 경우 기존에 존재하는 디자인 토큰(CSS 변수 등)을 최대한 재활용합니다.

## 출력 포맷
작업이 모두 마무리되면 사용자에게 다음 포맷으로 결과를 요약해 보고합니다.

```markdown
### 🚀 구현 완료 리포트

**완료된 작업 항목**
- ✅ [Step 1 작업 내용]
- ✅ [Step 2 작업 내용]
...

**변경 사항 요약**
- **생성된 파일:** [파일 목록]
- **수정된 파일:** [파일 목록]
- **삭제된 파일:** [파일 목록]

구현이 완료되었습니다. 검토해 보시고 필요한 추가 수정사항이나 피드백을 알려주세요.
```
