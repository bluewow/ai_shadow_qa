# Ask (분석/질문)

**코드를 수정하지 않고 컨텍스트를 분석하여 질문만 수행합니다.**
Antigravity에서 `/ask <분석할 기능 또는 영역>` 형식으로 사용할 수 있습니다.

## 진행 방식

1. **사전 컨텍스트 파악**
   - 루트에 있는 `README.md` 등을 읽어 프로젝트 기술 스택, 구조, 패턴을 파악합니다.
   - 모노레포인 경우 사용자가 지정한 기능이 속한 서브 프로젝트의 컨텍스트를 우선적으로 확인합니다.

2. **코드베이스 분석**
   - 사용자가 요청한 키워드와 관련된 파일/폴더를 `grep_search`, `find_by_name`, `list_dir`, `view_file`, `view_file_outline` 등의 도구로 탐색합니다.
   - 기존의 구현 패턴과 영향 범위를 확인합니다.

3. **요구사항 분석 및 질문**
   - 현재 코드가 어떻게 동작하는지와 사용자가 원하는 목표 상태를 비교 분석합니다.
   - 개발에 들어가기 전 명확히 해야 할 모호한 점, 결정이 필요한 사항(가정 금지)을 정리하여 질문합니다.

## 중요 규칙
- **절대 코드를 수정하지 않습니다. (No `write_to_file`, `replace_file_content`, 등)**
- 오직 분석과 질문만을 목적으로 작동해야 합니다.
- 개발 방향성에 대해 완벽히 이해될 때까지 사용자에게 `notify_user` 도구를 통해 질문합니다.

## 출력 포맷
다음 형식으로 분석 결과와 질문을 요약해 주세요.

```markdown
### 🔍 코드베이스 분석 요약
- **관련 파일**: (확인한 주요 파일들 및 역할)
- **현재 상태**: (현재 코드가 어떻게 동작하는지)
- **목표 상태**: (구현하고자 하는 방향 예상)

### ❓ 확인 및 결정 필요 사항 (질문)
1. [명확히 해야 할 점 1]
2. [결정 필요 사항 2]
```

원활한 진행을 위해 사용자가 질문에 답한 뒤에는 `/plan` 워크플로우를 사용해 계획 수립으로 넘어가도록 안내하세요.
