# Plan (계획)

**분석된 내용을 바탕으로 단계별 구현 계획(체크리스트)을 수립합니다.**
Antigravity에서 `/plan <계획할 기능>` 형식으로 사용하거나 `/ask` 완료 후 이어서 사용합니다.

## 진행 방식

1. **사전 컨텍스트 파악**
   - 사용자 요청 및 관련 `README.md` 등을 다시 한번 확인하여 컨벤션과 패턴을 숙지합니다.

2. **구현 계획 리스트업**
   - 수정/생성/삭제가 필요한 모든 파일의 절대 경로와 변경 내용을 리스트업합니다.
   - 선행 작업부터 테스트까지 논리적인 구현 순서를 Step 단위로 분할합니다.

3. **위험 요소 분석**
   - 예상되는 문제점, 사이드 이펙트, 주의해야 할 사항들을 명시합니다.

## 중요 규칙
- **실제 코드를 수정하지 않습니다.** 이 워크플로우는 `implementation_plan.md`와 같은 계획 문서 작성을 목표로 합니다.
- 프로젝트 내의 기존 패턴과 컨벤션을 최우선으로 따릅니다.
- 계획 수립이 완료되면 `notify_user` 도구를 사용하여 사용자의 승인을 받습니다.

## 출력 및 산출물 포맷
가능하다면 `implementation_plan.md` 아티팩트로 계획을 작성하거나 아래 포맷으로 응답하세요.

```markdown
### 📝 구현 계획

**변경 파일 목록**
- `[ADD]` 파일경로: 역할 설명
- `[MOD]` 파일경로: 변경 내용 요약
- `[DEL]` 파일경로: 삭제 이유

**구현 순서 (Step-by-Step)**
1. [Step 1 작업명]: 상세 작업 내용
2. [Step 2 작업명]: 상세 작업 내용
3. [Step 3 작업명]: 상세 작업 내용

**주의 및 확인 사항**
- [위험 요소 1]
- [고려 사항 2]

---
이 계획대로 진행할까요? 
승인하시면 `/go` 워크플로우를 통해 실제 코딩을 시작하겠습니다.
```
